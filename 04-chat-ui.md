# 模块说明：Chat UI

当前界面是一个标准的聊天工作台，而不是控制台或配置页。所有设计判断都围绕一件事：尽量缩短“提出需求 -> 得到文案”的路径。

## 布局结构

### 左侧

左侧是线程列表，只做三件事：

- 新建聊天
- 切换历史线程
- 删除线程

删除操作走 hover 后的 `…` 菜单，不直接在列表里放一个常驻关闭按钮。原因是后面这里还会继续长功能，菜单结构更稳。

### 右侧

右侧是主聊天区，分成三块：

- 顶部：最轻量的线程标题栏
- 中部：消息流与空态推荐
- 底部：输入框与发送控制

## 空态推荐

空态推荐不再是固定四句话，而是每次动态生成一组四条建议。  
右上角有一个“换一批”，可以刷新当前这组推荐。

当前策略：

- 首页默认只用简体推荐池
- 不主动展示繁体港风推荐
- 真正进入香港语境时，由后台判断切换，不在首页提前暴露

## 输入区

输入区是页面里最重要的控件，设计上要满足两个要求：

- 足够稳，像可以长期工作的地方
- 足够轻，不像一个堆满工具开关的面板

现在底部保留：

- 多行输入
- 回车发送提示
- 发送 / 停止按钮

前面试过把简繁切换塞进输入框，后来撤掉了。原因是这类模式不该常驻在前台。

## 消息区

当前消息区遵循一条简单原则：

- 用户消息更收束
- 助手消息更适合阅读

所以用户消息是偏紧的气泡，助手消息则是更舒展的白底内容块，方便读长文案。

## 反馈入口

每条助手消息下方有：

- `满意`
- `不满意`

这不是装饰按钮，而是 RAG 的入口。  
用户点“满意”后，这条内容会进入收藏样本池，影响后续召回。

## 当前交互上的几个重点

### 1. 线程删除

删除不只是本地删掉，还会调用 `/api/chat-thread`，同时清理 Supabase 里同一 `session_id` 的相关记录。

### 2. 推荐语刷新

“换一批”现在走轻量淡入，不做复杂转场。之前尝试过更强的动画，实际观感并不好。

### 3. 流式阶段的本地持久化

流式输出期间不会每个 chunk 都立刻把线程全量写回本地存储，而是做了延迟同步，减少卡顿。

## 当前不做的事

这版界面刻意不做以下内容：

- 单独的设置页
- 常驻模式切换
- 大量状态卡片
- 仪表盘式信息块

因为这些内容都会稀释聊天主链路，让页面更像后台系统，而不是文案工作台。
