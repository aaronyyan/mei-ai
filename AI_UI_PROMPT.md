# Mei AI UI 工作说明

这份文件给接手前端界面的开发者用。不是宣传文案，也不是给模型看的花哨 prompt。读完它之后，应该能知道这个项目的界面改动该往哪里收，哪些方向不要碰。

动手之前先看 `DESIGN.md`。

## 先记住这几个前提

- 这是医美文案工作台，不是通用 AI 聊天壳子
- 页面应以聊天为中心，不要做成后台系统
- 所有新增控件都要先问一句：它有没有打断“输入需求 -> 得到文案”的主流程

## 当前界面的固定原则

### 1. 聊天优先

左侧是线程，右侧是对话，底部是输入框。  
不要把页面改成信息卡片、统计卡片、设置面板拼起来的混合页。

### 2. 控件能藏就别挂满

不是所有状态都要露在前台。

例如：

- 删除对话放在 `…` 菜单里，而不是列表上挂一排按钮
- 简繁判断放在后台自动识别，而不是前台常驻开关

### 3. 空态推荐要像真实可点的工作入口

空态不是欢迎页，也不是营销区。  
它只负责一件事：帮用户更快开第一轮对话。

## 如果要改 UI，优先动这些地方

1. 布局层级
2. 留白和密度
3. 输入区质感
4. 消息阅读舒适度
5. 线程列表的清晰度

先改结构，再改颜色；先改关系，再改装饰。

## 明确不要做的事

- 不要堆过多图标按钮
- 不要把 header 塞满模式、状态和辅助操作
- 不要让空态像运营活动页
- 不要把聊天页面改成 dashboard
- 不要为了“高级感”上大量无意义动画

## 适合的改动方式

如果你要给另一个开发者或模型一句简短说明，用下面这句就够：

让它更像一个成熟的聊天工作台：线程清楚、输入区稳、消息更好读、控件更克制，不要有后台系统气。
